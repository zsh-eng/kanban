## Main idea

Task tracking can be separate from the dev log.

Dev log can just be a stream of thoughts / learnings (we can also create a simple way of managing this)
The task tracking will live within a single markdown file, without too much additional metadata.

Mark task as complete --> prepend checkbox and also preprend date of completion.
E.g. `[x] 2025-03-31: Task title`
This allow us to easily parse and group the completed tasks by date / render out a view later on in filterable table view.

Any additional information about the task can just.

"CURRENTLY WORKING ON" - can either be in the dev log or just the top of the kanban board.
